Getting Started.

Floe is a small, simple and elegant workflow engine. Floe's main intention is to provide a continuous delivery tool favouring code over config.

It is made up of a shared library *floe* and your own custom workflows in a folder called *customfloe*

Customfloe code takes the place of the giant XML files you see in traditional CI tools

Quick start

To set up a localised self contained go development environment, read on (to include flow in your own go workspace read further ahead)

Clone this repo (or your fork) locally, then make create and build e.g.

git@github.com:floeit/floe-build.git
make create
make build

and run it...

bin/workflow-agent

Now point your browser at:

http://localhost:3000/build

Manual Setup

  • Fork this repo https://github.com/floeit/customfloe
  • Create a repo to house your custom workflows
  • Clone the repos into your go environment *src* folders
  • run *go get* and *go install* as normall
  • Fork this repo https://github.com/floeit/public
  • clone it locally at the same level as where you are running the binary from
  • run the binary e.g. bin/workflow-agent
